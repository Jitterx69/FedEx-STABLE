To achieve the "Most Secured" status for your CI/CD pipeline, you must manually configure the following settings in your GitHub Repository (`https://github.com/Jitterx69/FedEx-STABLE`).

## 1. Branch Protection Rules
**Go to:** `Settings` > `Branches` > `Add rule`

**Pattern:** `main` (and `develop` if desired)

**Check compliance:**
*   [x] **Require a pull request before merging**
    *   [x] Require approvals (Recommended: 1 or 2)
*   [x] **Require status checks to pass before merging**
    *   Search for and select your workflow jobs:
        *   `Security Audit`
        *   `Frontend CI`
        *   `Backend (Rust) CI`
        *   `ML (Python) CI`
        *   `Analyze (javascript-typescript)`
        *   `Analyze (python)`
*   [x] **Require conversation resolution before merging**
*   [ ] (Optional) **Do not allow bypassing the above settings** (Enforce even for admins)

## 2. Secrets and Variables
**Go to:** `Settings` > `Secrets and variables` > `Actions`

If your application deploys to a server or uses external APIs (like AWS, Database URLs), **NEVER** commit them to code.
*   Click **New repository secret**.
*   Add secrets like:
    *   `PROD_DB_URL`
    *   `AWS_ACCESS_KEY_ID`
    *   `DOCKER_HUB_TOKEN`

## 3. General Security Settings
**Go to:** `Settings` > `Code security and analysis`

*   [x] **Dependabot alerts**: Enabled
*   [x] **Dependabot security updates**: Enabled
*   [x] **Secret scanning**: Enabled (if public repo) or Enabled (if GHAS is active)
*   [x] **Code scanning**: Set up (We added `codeql.yml` for this!)
